📋 DSL Format Specification

  • Indentation-based: Use spaces for nesting (recommended: 4 spaces per level)
  • Element syntax: type, name: type, or name: type props...
  • Triple dash (---): Marks end of props section
  • Children: Lines after --- with greater indentation
  • Props format: key: value or key value1 value2...
  • Special arrays: at 0 0 → "at": [0, 0]
  • Directives: Start with @ (e.g., @layout)
